Lambda表达式

介绍

Lambda表达式是基于函数式编程的思想，支持将代码片段作为方法参数进行使用。可以理解为一种更加简洁的可传递匿名函数的书写方式。

Lambda表达式没有名称，而且不属于某一个类，但是可以有参数列表、代码体、返回值。

当在Lambda方法体内使用外部变量时，其必须声明为final。

使用规范

(参数列表) ‐> {
    方法体
}

-> ：Lambda运算符

方法体：用于执行业务逻辑。可以是单一语句，也可以是语句块。如果是单一语句，可以省略花括号。当需要返回 值，如果方法体中只有一条语句，可以省略return，会自动根据结果进行返回。

没有参数、一条语句的Lambda表达式

()‐>new Student();

一个参数、多条语句的Lambda表达式

x ‐> {
    System.out.println(x);
    return x; 
}

多个参数、一条语句的Lambda表达式

(x,y) ‐> x+y;

在Lambda中，JVM会根据上下文推断参数列表中参数的数据类型，不用定义类型。

底层原理

在编译时，Lambda表达式的内容会被编译为所在类的静态私有方法，在执行时，对Lambda表达式的调用会进入LambdaMetafactory类的metafactory中，该方法会为Lambda表达式创建内部类，并调用Lambda编译后的方法。

函数式接口

函数式接口是仅有一个抽象方法的普通接口，为了让现有的函数能够更加友好的使用Lambda表达式。可以通过在接口上添加@FunctionalInterface注解来自定义一个函数式接口，函数式接口中如果声明多个抽象方法则会报错，但是默认方法和静态方法在此接口中可以定义多个。

在Java8的类库设计中，已经引入了几个函数式接口：Predicate、Consumer、Function、Supplier

  • Predicate：Predicate接口是Java8定义的一个函数式接口，属于java.util.function包下，用于进行判断操作，内部定义一个抽象方法test、三个默认方法and，negate，or、一个静态方法isEqual。test()方法接收一个泛型T对象，返回boolean。
  • Consumer：Consumer也是JDK8提供的函数式接口，用于进行获取数据的操作，其内部定义了一个抽象方法accept、一个默认方法andThen。accept()方法接收一个泛型T对象，无返回值。
  • Function：Function主要用于进行类型转换的操作。内部提供一个抽象方法apply、两个默认方法compose，andThen、一个静态方法identity 。apply()方法接收一个泛型T对象，并返回一个泛型R的对象。
  • Supplier：Supplier用来进行值获取操作，内部只有一个抽象方法get，返回一个范性T的对象。
  • UnaryOperator：扩展至Function<T,T>，所以这个本质上也是一个映射转换接口，只不过映射转换后的类型保持不变
  • BiFunction：包含apply方法，接受泛型的T、U，返回R，可以视为复合型映射转换接口
  • BinaryOperator：扩展至Function BiFunction<T,T,T>，所以这个本质上也是一个复合型映射转换接口，只不过映射转换后的类型保持不变
  • BiPredicate：包含test方法，接受泛型的T，U，返回boolean，可以视为复合型断言（检查）接口
  • BiConsumer<T,U>：包含accept方法，接受泛型的T，U，无返回，可以视为复合型数据消费接口
